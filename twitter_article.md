# 我用 MiroFish 重构了 BTC 恐慌指数：500 个 AI 投资者模拟恐慌传染，传统指数该淘汰了

传统的 Crypto Fear & Greed Index 已经连续 38 天显示"极度恐惧"，分数低至 18。每次 BTC 大跌，所有人都盯着这个数字。但很少有人问过一个基本问题：**这个数字是怎么算出来的？它还可靠吗？**

我花了一周时间拆解它，发现它 40% 的权重来自三个几乎失效的数据源。然后我用 MiroFish 群体智能引擎彻底重构了这部分，用 500 个 AI 投资者在模拟社区里真实互动，来捕捉传统指数完全忽略的东西——**恐慌的传染路径**。

---

## 一、传统恐慌指数的致命缺陷

先看构成。Alternative.me 的 Fear & Greed Index 由六部分组成：波动性(25%)、市场动量(25%)、社交媒体情绪(15%)、调查问卷(15%)、BTC 市值占比(10%)、Google Trends(10%)。

波动性和动量没问题，这是硬数据。但后面四项加起来占 50% 权重，全都有严重问题：

**调查问卷** — alternative.me 自己已经暂停了这个维度。样本量小，数据滞后，形同虚设。

**Google Trends** — 搜索"Bitcoin"的人可能是恐慌想卖的，也可能是好奇想买的，还可能是写作业的学生。一个搜索量数字，无法区分情绪方向。

**社交媒体情绪** — 这是最关键的一项，也是烂得最彻底的一项。

传统指数用 VADER（一个 2014 年发表的通用英文情绪分析工具）来扫描推文和 Reddit 帖子。我用真实的 Reddit r/Bitcoin 帖子测试了一下 VADER 在加密语境下的表现：

| 帖子内容 | VADER 打分 | 判定 | 实际情绪 |
|---------|-----------|------|---------|
| "My Mom told me she wants to buy Bitcoin" | 98.1 | 极度贪婪 | 中性（个人分享） |
| "Daily Discussion Thread - March 2026" | 96.4 | 极度贪婪 | 中性（模板标题） |
| "Made profit $3 from this bull run!" | 100.0 | 极度贪婪 | 讽刺/自嘲 |
| "Where to buy btc?" | 93.0 | 极度贪婪 | 中性（新手提问） |
| "finally hit my first whole coin" | 97.8 | 极度贪婪 | 中性（个人里程碑） |

VADER 把所有包含正面词汇（great, good, happy, love, buy）的帖子一律标记为"贪婪"。在我测试的 100 条 r/Bitcoin 帖子中，**36% 被判为"极度贪婪"** — 而同一时间传统恐慌指数显示的是 23 分"极度恐惧"。

一个连讽刺都读不懂、把"Daily Discussion Thread"当成极度贪婪信号的模型，凭什么给你判断市场情绪？

更深层的问题是：即便情绪分析准了，**统计聚合也不等于理解传播**。知道"30% 的帖子是恐慌情绪"远远不够，你需要知道的是：这些恐慌帖子有没有在传染？被谁看到了？散户看到 KOL 喊崩盘后反应如何？恐慌是在放大还是在被对冲？

传统指数回答不了这些问题。它只是把一堆数字做加权平均，输出一个 0-100 的分数。

## 二、MiroFish 改造方案：两阶段架构

MiroFish 是一个开源的群体智能模拟引擎，核心理念是：创建大量具有独立人格和记忆的 AI Agent，让它们在模拟社区中自由交互，通过群体行为的涌现来做预测。

它的底层引擎 OASIS（由 CAMEL-AI 开发）可以模拟 Twitter 和 Reddit 平台 — Agent 可以发帖、评论、点赞、踩、关注。这不是让一个 LLM 扮演 8 个角色轮流发言，而是真正的**多 Agent 并行交互**，每个 Agent 独立决策，看到不同的帖子，做出不同的反应。

我的改造分两个阶段：

### Stage 1：全量 LLM 情绪分析

从三个平台实时采集社交数据：
- **币安广场**（Playwright 无头浏览器绕过 WAF）— 中文投资者最真实的恐慌声音
- **Twitter/X**（通过 Nitter 开源前端）— 英文主流加密讨论
- **Reddit**（14 个子版块的 RSS + 搜索）— 深度讨论和分析帖

一次采集约 2800 条帖子。不用 VADER，**全部发给 DeepSeek 做加密语境感知的情绪分析**。每 40 条一批，4 线程并行，2 分钟分析完全部数据。

DeepSeek 理解加密语境：
- "Buy the dip" 在暴跌 30% 时 → 恐惧性抄底（偏恐慌），不是贪婪
- "To the moon" 在横盘期 → 无聊灌水（中性），不是极度贪婪
- "合约全爆仓，恐慌性抛售" → 真正的极度恐惧
- "Daily Discussion Thread" → 50 分中性，而不是 VADER 给的 96 分贪婪

每条帖子还按参与度加权：一条 10,000 赞的恐慌帖代表的群体情绪，显然比一条 1 赞的个人牢骚重要得多。

Stage 1 输出的不是一个简单分数，而是**完整的情绪分布**：极度恐惧占比、恐惧占比、中性占比、贪婪占比、极度贪婪占比，以及分平台的情绪差异。

### Stage 2：500 Agent 群体智能模拟

这是核心创新。

我用程序化方式生成了 500 个差异化的 BTC 投资者 Agent，按照真实市场参与者分布：

| 类型 | 数量 | 特征 |
|------|------|------|
| 高恐慌散户 | 120 | ATH 附近入场，亏损中，每 5 分钟看一次盘 |
| 中等散户 | 80 | 有些经验，但仍受社交媒体影响 |
| 活跃交易者 | 60 | 技术分析，日内/波段交易 |
| 杠杆交易者 | 40 | 5-50x 杠杆，对价格极度敏感 |
| 长期持有者 | 50 | 2017 年入场，经历过多次 -80% |
| DeFi 用户 | 35 | 用 BTC 做抵押，关注清算线 |
| 机构投资者 | 25 | 数据驱动，反向操作 |
| KOL/网红 | 25 | 放大情绪，影响大量粉丝 |
| 矿工 | 20 | 关注挖矿成本和电费 |
| 纯新手 | 45 | 上个月从朋友口中听说 BTC |

**每个 Agent 不是模板复制**。同样是"散户"，Agent #37 在 $68K 入场亏了 8%，性格 ESFP，每 15 分钟看盘，主要受 YouTube 博主影响；Agent #102 在 $71K 入场亏了 4%，性格 ISFP，每小时看盘，主要受 Reddit 社区影响。他们对同一条恐慌帖的反应会不同。

入场价格、仓位占储蓄比例、风险偏好、信息来源、国籍、性格（MBTI）、年龄 — 全部随机化。500 个 Agent，500 种不同的人。

然后我把 Stage 1 的结果喂进这个模拟社区：
1. 先发一条统计摘要帖（"2800 条帖子分析：极度恐惧 X%，恐惧 Y%…"）
2. 再 seed 50 条有代表性的真实帖子（从情绪分布中分层抽样，确保恐慌帖和乐观帖都有）

然后放手。500 个 Agent 在模拟 Reddit 上自由交互 5 轮。每轮每个 Agent 看到推荐的帖子，独立决定：发帖？评论？点赞？踩？还是什么都不做？

5 轮下来，产生数千条交互。

### 关键发现：恐慌传染效应

在一次模拟中，我观察到了教科书级的恐慌传染路径：

1. **KOL Agent** 发了一条 "$29K 崩盘预测" 的帖子（这是从真实 Twitter 数据中 seed 进来的）
2. **Newbie Agent** 看到后立刻恐慌，发帖 "OMG should I sell everything??"
3. **Retail Agent** 看到 KOL 的预测 + Newbie 的恐慌，开始动摇
4. **更多 Retail Agent** 看到越来越多的恐慌帖，信心进一步瓦解
5. 甚至 **Macro-Neutral Agent**（最理性的分析型 Agent）在第 4 轮也被感染，开始转述 $29K 的叙事

**只有 Diamond Hands（2017 年入场的老韭菜）和 Whale（机构投资者）全程保持冷静。**

Claude 分析模拟结果后给出的评估：contagion_effect = "amplified" — 恐慌被群体互动放大了。

**这就是 OASIS 多 Agent 模拟的真正价值。** 它不是情绪分析的替代品，它做的是传统指数完全无法做的事情 — **模拟情绪在不同类型投资者之间的传播和放大路径，预测恐慌会不会级联，以及谁在扛住、谁在崩溃。**

## 三、技术实现

完整技术栈（全部开源/免费或极低成本）：

**数据采集层**
- 币安广场：Playwright 无头浏览器，绕过 AWS WAF，遍历 7 个 hashtag，深度滚动加载
- Twitter/X：通过 Nitter 开源实例，20 个关键词搜索 + 分页
- Reddit：14 个子版块 × 4 种排序（hot/new/rising/top）的 RSS feed + 9 个搜索词

**Stage 1：情绪分析**
- DeepSeek API 批量打分（40 条/批，4 线程并行）
- 加密语境感知 prompt（区分真实情绪 vs 日常讨论）
- 参与度对数加权（赞/评论/转发）

**Stage 2：群体模拟**
- OASIS 引擎（camel-oasis）
- 500 个程序化生成的差异化 Agent
- DeepSeek 驱动 Agent 决策（成本 < $1/次模拟）
- Claude 分析模拟输出并评估传染效应

**指数计算**
- 波动率 25%（CoinGecko 30 日价格，重新校准映射函数）
- 市场动量 25%（价格 vs 30 日均值 + 24h 变化）
- BTC 市值占比 10%
- MiroFish 情绪 40%（Stage1 统计分 × 0.4 + Stage2 模拟分 × 0.6）

LLM 调用走 Claude Max 的 OAuth（通过 claude CLI，不需要 API Key）和 DeepSeek（几毛钱 100 万 token）。完全没有用 MiroFish 原版依赖的 Zep Cloud 付费服务，直接调 OASIS 底层引擎。

## 四、一个行业级问题：数据饥荒

做完这个项目后，我最大的感受不是技术难度，而是**数据源的匮乏**。

我从三个平台爬了约 2800 条帖子。这够不够？不够。这个数字至少需要放大 100 倍到 28 万条才有足够的统计显著性。

但问题出在供给侧：

**币安广场** — 没有公开 API。我必须用 Playwright 模拟浏览器、绕过 AWS WAF 反爬机制才能拿到数据。每次只能爬到 300-400 条，而且不稳定，经常超时。币安广场上有全球最大的加密社区讨论数据，却被锁在围墙花园里。

**Twitter/X** — 免费 API 限制为每 15 分钟 1 个请求，形同虚设。基础付费套餐 $100/月，企业版 $42,000+/月。我被迫使用 Nitter（第三方开源前端），但 Nitter 实例随时可能被封杀。

**Reddit** — 2023 年封杀了免费 JSON API 访问。只能走 RSS feed，数据量有限，没有评论内容，没有投票数。

三个平台，三种不同的限制方式，但结果一样：**开发者拿不到数据。**

## 五、给交易所的呼吁

加密行业天天在说"去中心化"和"开放透明"。但最大的交易所们，把最有价值的社区情绪数据锁在自己的围墙花园里。

我的诉求很具体：

**1. 开放广场/社区帖子的公开 API**

不需要用户隐私信息。只需要：帖子文本 + 时间戳 + 点赞数 + 评论数。设置合理的频率限制就行。

**2. 提供聚合情绪数据**

交易所内部一定有情绪分析系统。把聚合后的情绪分布（每小时/每天的恐慌-贪婪分布）作为公开数据发布。类似于你们已经在做的恐慌指数页面，但开放原始数据。

**3. 支持开发者生态**

设立 Developer Program，给研究项目和开源项目免费 API 额度。你们花几百万赞助黑客松，但开发者连最基本的社区数据都拿不到。

如果币安愿意开放广场 API，仅这一个平台就能让我们的数据量从 300 条跃升到 30,000+ 条。如果加上 OKX、Bybit、Bitget，轻松突破 10 万条。到那个量级，MiroFish 模拟才能真正发挥群体智能的统计优势。

## 结语

传统的 Crypto Fear & Greed Index 诞生于 2018 年，核心方法论 10 年没有实质性进化。VADER 还在用，调查问卷已经停了，社交数据的处理方式停留在关键词统计时代。

我们现在有 LLM 能理解语境和讽刺，有 OASIS 这样的多 Agent 模拟引擎能模拟群体行为的涌现，有足够便宜的 API 让 500 个 AI 投资者跑一次模拟只需要不到 1 美元。

**工具已经准备好了。缺的只是数据。**

项目完全开源：github.com/robin-ph/btc-fear-greed-index

`python main.py` 一键运行。欢迎 PR，特别需要：更多数据源（韩国论坛、日本论坛、Telegram）、更好的 Agent 人设、历史回测验证。

感谢 MiroFish 团队的开源引擎。一个大四学生的毕业设计，GitHub 全球趋势第一，拿到 3000 万投资。我只是站在他的肩膀上，把 OASIS 用到了一个具体的金融场景。

这就是开源的力量。
